# 💖 愿此见闻温柔你的日常

<div align="center">

**在恰好的时刻，循着时间、生活状态、聊天记忆与网络热搜，把一路拾来的所见所闻化作文字、语音、配图或视频，分享到私聊、群聊或 QQ 空间，为平凡日常添一点新鲜与温柔。**


</div>

## 📖 简介

这是一个给角色补上“日常生活分享”的插件。Ta不再只等待一句问题才回应，而是会循着时间、天气、生活状态、聊天记忆和网络热搜，在合适的时刻把所见所闻分享到你身边。

清晨可以是一句带着阳光的早安语音，午后可以是一张热搜长图，深夜可以是一段安静的心情视频，也可以自动写成 QQ 空间里的日常说说。Ta关心的不只是“分享出去”，而是让每一次分享都更自然、更像正在生活。

## 🌟 核心能力

### 1. 🔍 真实来源：联网检索增强 (Tavily / Brave)

- 新闻分享优先使用新闻源 API 自带摘要、正文和链接；背景不足时再调用 AstrBot 配置的 Tavily 或 Brave 搜索。
- 知识和推荐会并发参考百科与联网检索，降低模型凭空发挥的概率。

### 2. 📝 内容分享

- 支持 `问候`、`新闻`、`心情`、`知识`、`推荐` 五类日常分享。
- 支持微博、知乎、B站、抖音、快手、头条、百度、腾讯、小红书、夸克、36氪、51CTO、A站、爱范儿、网易、新浪、澎湃、第一财经、财联社等热搜源。
- 支持每天 60s 读世界和 AI 资讯快报长图。
- 支持自定义知识库、推荐库和类型前缀显示。
- 支持新闻长图原文链接自然语言追问，例如“刚才热搜图第 3 条链接”。

### 3. 🎨 多媒体

- 配图、视频和语音默认调用 `astrbot_plugin_daily_life`。
- 视频使用配图生成，图片和视频不会重复混发。
- 语音失败不会阻断文案、配图或视频发送。
- QQ/OneBot 群聊和私聊会优先把远程图片下载到本地后发送，提升富媒体发送稳定性。
- 个人微信图片会生成发送副本，并按数量自动清理临时文件。

### 4. 🌐 QQ 空间

- 支持发布文字、图片或单视频说说。
- 支持查看我的说说、指定 QQ 空间动态、说说详情、点赞、评论和删除自己的说说。
- 支持 QQ 空间独立定时、独立类型序列和智能定时。
- 支持在多机器人实例中为 QQ 空间单独选择实例；留空时默认使用当前缓存的第一个实例。
- 支持好友动态自动点赞、好友动态自动评论、好友在 Bot 评论楼层下回评后的自动续评。
- 支持自己说说下的好友评论自动回评，并通过落点校验尽量避免回错楼层。
- 支持好友动态配图识别，可指定视觉模型；识别失败时自动按纯文字动态评论。
- 支持合并好友动态和“与我相关”入口，提高好友说说提及 Bot 或回评后的命中率。

### 5. ⏰ 调度

- 支持固定时间、随机时段、智能定时和高级 Cron。
- 高级 Cron 支持 5 位、6 位和 7 位格式。
- 支持群聊、私聊独立定时和独立类型序列。
- 随机时段和智能定时生成的任务会写入数据库，重启后可恢复未完成任务。

### 6. 🧠 上下文

- 可读取生活状态、聊天历史和群聊热度，让分享内容更贴近当前场景。
- 主动分享写入历史时使用内部标记，后续读取时会降级为背景信息，避免污染普通对话。
- 可复用 `astrbot_plugin_daily_life` 的 MemOS，把分享记录写入长期记忆。
- 插件未单独指定 LLM 时，会优先使用 AstrBot 当前会话的模型配置。

### 7. 🖥️ 仪表盘

- 状态总览：查看自动分享、当前时段、接收目标、定时任务和生成进度。
- 手动触发：选择分享目标、分享类型和新闻源。
- 动态与媒体：查看今日分享、文案、图片、视频和历史记录。
- QQ 空间面板：查看动态、发布说说、点赞、评论、删除和查看详情。

## 🚀 快速开始

1. 在 AstrBot 插件市场安装 `astrbot_plugin_daily_share`。
2. 如需配图、视频、语音、生活状态或 MemOS 长期记忆，安装并配置 `astrbot_plugin_daily_life`。
3. 在目标私聊或群聊发送 `/sid`，复制 `UID` 或 `Session ID`。
4. 在当前会话发送 `/分享 添加当前`，加入每日分享接收对象。
5. 发送 `/分享 心情` 或 `/分享 新闻` 测试当前会话发送。
6. 打开 AstrBot WebUI 插件页面，进入每日分享仪表盘调整配置。

## 🧭 平台能力

| 平台 | 支持场景 | 接收对象填写 | 说明 |
| :--- | :--- | :--- | :--- |
| QQ / OneBot (`aiocqhttp`) | 私聊、群聊、QQ 空间 | 私聊填 QQ 号，群聊填群 `Session ID` | QQ 空间需要适配器支持 `get_cookies`；多实例可在 `qzone_conf.qzone_adapter_id` 指定 |
| 个人微信 (`weixin_oc`) | 个人私聊 | 微信 openid 或 `Session ID` | 不支持群聊；主动发送需要 AstrBot 已保存 `context_token` |
| WebChat / 其Ta AstrBot 平台 | 取决于适配器 | 优先使用 `/sid` 输出 | 主动发送能力取决于适配器 |

个人微信专属逻辑只对平台名或适配器元信息精确为 `weixin_oc` 的适配器启用。

## 🎯 目标配置

接收对象只填写 `/sid` 输出的纯 `UID` 或 `Session ID`，不要填写完整 UMO。

正确示例：

```text
123456
o2cq508wSbYFvbrV5tnC29LoU3NU@im.wechat
123456:心情,新闻
123456:08:30:问候,心情
123456:0 8 * * *:新闻,心情
```

错误示例：

```text
default:FriendMessage:123456
default:FriendMessage:o2cq508wSbYFvbrV5tnC29LoU3NU@im.wechat
```

配置格式：

- `纯 ID`：跟随全局定时。
- `纯 ID:类型序列`：跟随全局定时，但只按指定类型轮换。
- `纯 ID:时间:类型序列`：使用独立定时和独立类型序列。
- 支持类型：`问候`、`新闻`、`心情`、`知识`、`推荐`。

## 💬 自然语言触发

开启 AstrBot LLM 工具调用后，可以用自然语言触发分享。当前会话需要是管理员或已配置接收对象。

| 意图 | 示例 | 动作 |
| :--- | :--- | :--- |
| 发空间说说 | `今天天气不错，分享下空间说说吧` | 生成并发布 QQ 空间说说 |
| 发空间视频 | `把今天的心情做成空间视频` | 生成配图和视频后发布到 QQ 空间 |
| 发心情语音 | `分享下心情，念给我听` | 生成文案和语音 |
| 看热搜长图 | `看看微博热搜的图片版` | 发送指定新闻源长图 |
| 查新闻链接 | `刚才热搜图第 3 条链接发我` | 从缓存中返回原文链接 |
| 发早报 | `给我来一份 60s 新闻` | 发送每天 60s 读世界长图 |

## 🌐 QQ 空间自动互动

QQ 空间自动互动由 `qzone_conf.qzone_enable_auto_interaction` 统一控制，开启后会按同一个查询间隔或 Cron 任务执行已启用的子能力。

- 自动点赞：扫描好友动态，跳过自己的动态和已处理动态。
- 自动评论：扫描好友动态，由 LLM 生成自然短评论；动态带图时可先进行配图识别。
- 好友动态续评：Bot 已在好友说说下评论后，如果好友继续在该楼层回评，Bot 会尝试继续续评。
- 自己说说回评：扫描自己说说下的好友评论，生成回复并尽量校验回评落点。
- 配图识别：默认识别 1 张，最多 9 张；可指定视觉模型，留空或失败时按纯文字动态评论。
- 多实例：`qzone_adapter_id` 只影响 QQ 空间发布、查询、点赞、评论和自动互动，不改变普通群聊/私聊发送路由。

## 🧯 常见问题

### Q1：为什么定时任务无法主动发送个人微信？

`weixin_oc` 主动发送需要 AstrBot 保存过该 openid 的 `context_token`。请让该微信私聊先给机器人发任意消息，再等待下一次定时任务或手动触发一次。

### Q2：个人微信支持群聊吗？

不支持。`weixin_oc` 个人微信适配器目前仅支持个人私聊。

### Q3：为什么完整 UMO 会被跳过？

接收对象只使用 `/sid` 输出的纯 `UID` 或 `Session ID`。完整 UMO 容易导致平台判断错误，因此不会作为有效目标。

### Q4：QQ 空间 Cookie 过期怎么办？

插件会在发布失败时尽量清理内置登录态缓存，并重新向 OneBot 获取 `user.qzone.qq.com` 登录态后重试一次。若仍失败，请检查 OneBot 登录状态和 `get_cookies` 支持情况。

### Q5：AstrBot 有多个机器人实例时，QQ 空间默认使用哪个？

如果 `qzone_conf.qzone_adapter_id` 留空，QQ 空间会默认使用当前缓存的第一个机器人实例。需要固定使用某个 QQ 实例时，请在仪表盘 QQ 空间设置里选择对应实例。

### Q6：QQ 空间提示 `network busy` 或 `使用人数过多，请稍后再试` 是什么？

这是 QQ 空间接口临时繁忙或限流，不代表插件配置一定错误。自动互动会跳过本轮，等待下次定时触发再尝试。

### Q7：好友说说配图识别为什么没有生效？

请先开启 QQ 空间自动评论配图识别，并选择支持图片输入的视觉模型。未选择、模型不可用或识别失败时，插件会自动降级为纯文字动态评论。

### Q8：为什么 QQ 群/私聊图片要先下载到本地？

部分 QQ 适配器发送远程 CDN 图片不稳定。插件会把图片下载到插件数据目录的 `Temp` 后再发送，以提升送达率。

### Q9：分享写入聊天历史会污染普通聊天吗？

不会。插件会用内部标记记录主动分享，读取历史时会把Ta降级为背景信息，不会伪装成用户真实请求。

### Q10：仪表盘为什么使用轮询？

AstrBot Pages 环境下轮询更稳定。仪表盘空闲时低频刷新，分享中提高刷新频率；QQ 空间和分享动作也会推送轻量事件用于即时更新。
